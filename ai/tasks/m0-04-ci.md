# Task: CI-Pipeline mit Path-Filtern (M0-04)

## Ziel
GitHub Action, die pro Monorepo-Modul nur das baut und testet, was sich geändert hat.
Deployment macht weiterhin Railway/Vercel über die Git-Integration – die CI ist reines
Sicherheitsnetz vor `main`.

## Kontext
- Monorepo `bfsg-monitor` mit `api/` (Maven), `web/` (Angular), `worker/` (Node/TS)
- M0-01 bis M0-03 sind abgeschlossen, alle drei Module bauen lokal
- Referenz: CI-Setup aus Kommuvo (GitHub Actions) als Vorlage für den Maven-Teil

## Anforderungen
1. Workflow `.github/workflows/ci.yml`, Trigger: push auf main + pull_request
2. Path-Filter (z. B. via `dorny/paths-filter` oder `on.push.paths` mit drei Workflows –
   architect entscheidet, einfachste wartbare Variante gewinnt):
   - Änderung in `api/**` → Job `api`: JDK 21 Setup, `mvn verify` (mit Testcontainers,
     Docker ist auf GitHub-Runnern verfügbar)
   - Änderung in `web/**` → Job `web`: Node 22, `npm ci`, `ng build`, Unit-Tests headless
   - Änderung in `worker/**` → Job `worker`: Node 22, `npm ci`, Typecheck + Tests
3. Maven- und npm-Caching aktivieren (Build-Minuten sparen)
4. Branch-Protection-Hinweis in README ergänzen: main nur mit grüner CI mergebar
   (die Einstellung selbst macht Lukas im GitHub-UI)

## Akzeptanzkriterien
- [ ] Push mit Änderung nur in `web/` triggert NUR den web-Job
- [ ] Push mit Änderung in allen drei Modulen triggert alle drei Jobs
- [ ] Alle Jobs laufen auf dem aktuellen Stand von main grün durch
- [ ] Zweiter Lauf ist durch Caching messbar schneller

## Nicht in Scope
- KEIN Deployment über die Action (Railway/Vercel deployen selbst)
- KEINE Release-/Tagging-Logik, keine Docker-Registry-Pushes
- KEINE E2E-Tests (kommt frühestens nach M2)
- KEIN Dependabot/Renovate-Setup (separat, später)

## Agent-Reihenfolge
1. architect → entscheidet Path-Filter-Variante (1 Workflow mit Filter vs. 3 Workflows)
2. implementer → Umsetzung
3. reviewer → Fokus: keine Secrets in Logs, Cache-Keys korrekt, Filter decken
   Randfälle ab (Änderung an Root-Dateien wie ai/architektur.md triggert nichts)
